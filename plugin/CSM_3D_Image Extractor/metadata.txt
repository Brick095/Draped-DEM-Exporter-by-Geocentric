about=Image Extractor automates the process of clipping a basemap (e.g. Google Satellite) and a DEM raster to a user-defined AOI polygon, producing georeferenced outputs ready for 3D texture draping workflows. Developed by Geocentric Environmental Inc. to streamline internal DEM and texture draping processes.
 
 The tool renders the visible map canvas to a high-resolution image, applies a precise vector mask from the selected AOI layer, auto-crops to the opaque pixel boundary, and clips the DEM to the exact same geographic extent. Both outputs share identical spatial extents, eliminating skirt artifacts in 3D visualization software.
 
 Outputs:
 - Tile__0__0.png — masked and cropped basemap image (PNG)
 - [Output Name].tif — DEM clipped to matching extent (GeoTIFF)
 
 Inputs configured through a simple dialog:
 - Basemap layer (raster)
 - DEM layer (raster)
 - Cropping Extent layer (vector polygon)
 - Output folder and file name
 - Output CRS — recommended WGS84 (EPSG:4326) for broadest compatibility
 - Cropping layer CRS — CRS of the vector layer used for cropping
 - Basemap CRS — CRS of the basemap imagery (Google Satellite is typically Web Mercator, EPSG:3857)
